# York Living Dashboard

Interactive customer dashboard for the York Living real estate project in Muenster.
Single-page Vite + TypeScript app with investment calculation, liquidity projection, and PDF export.

## Project Structure

- `dashboard/` - Main web app (Vite + TypeScript, no framework)
- `dashboard/src/main.ts` - All UI and calculation logic (monolith, see ROADMAP.md for split plan)
- `dashboard/src/style.css` - Stylesheet with CSS custom properties
- `dashboard/src/data/calculation-config.json` - All calculation parameters, apartment data, tax brackets
- `dashboard/public/` - Static assets (floorplans, hero images)
- `ROADMAP.md` - Architecture evolution plan
- `PROJECT_MEMORY.md` - Project decisions and scope
- `skills/` - Shared AI skills (finanz-audit, mlp-design-guide) - committed so all collaborators have them
- `deploy.ps1` - Build + SFTP deployment wrapper

## Commands

- Dev server: `cd dashboard && npm run dev`
- Build: `cd dashboard && npm run build` (builds with `--base=/YorkLiving/`)
- Deploy: `.\deploy.ps1` or `.\deploy.ps1 -SkipBuild`

## Conventions

- All calculation inputs are centralized in `calculation-config.json` - never hardcode values
- Verify `calculation-config.json` before changing any calculation logic
- UI and calculation are still coupled in `main.ts` - be aware of this when editing
- 8px spacing grid, body text >= 16px, mobile-first responsive
- Workspace boundary: only `D:\AppEntwicklung\Immobilien` - never touch `D:\AppEntwicklung\KundenPraesentationYorkQuartier`

## Design System

Two design guides are available:

### Default: Generic Design Guide (`SKILL.md`)
The app currently uses a neutral gray palette with a single accent color. See `SKILL.md` at repo root.

### MLP Corporate Design (on request)
When the user asks for MLP-branded design or mentions "MLP Design", "MLP CI", "MLP Farben", "MLP Corporate Design", or "MLP-konform":
1. Read the full reference: `skills/mlp-design-guide-v3/references/MLPDesignGuideV3.md`
2. Apply MLP brand colors: primary `#033D5D`, titanium `#BEB6AA`, text `#2B2B2B` (never pure black)
3. Follow WCAG accessibility rules from the guide
4. Use subbrand colors (MLP Med / SOFE) only when the audience requires it
5. Include a short compliance note in the output

## Finanz-Audit Skill (on request)

When the user asks to verify calculations or mentions terms like "rechnet die App richtig", "Plausibilitaetscheck", "Steuer-Check", "Denkmal-AfA pruefen", "Finanzmathe-Audit", "koennen wir das so dem Kunden zeigen":

1. Read the skill definition: `skills/finanz-audit-skill/SKILL.md`
2. Read reference files from `skills/finanz-audit-skill/references/`:
   - `steuer-regeln.md` - ESt-Tarif, Soli, KiSt, WK bei V+V
   - `afa-regeln.md` - Lineare AfA, Denkmal-AfA SS7i/SS7h/SS10f, Kaufpreisaufteilung
   - `finanzierung-regeln.md` - Annuitaeten, Zins-/Tilgungsplan, KfW-Besonderheiten
   - `referenzbeispiel-denkmal.md` - Vollstaendige Musterberechnung als Pruefschablone
3. Follow the 4-phase audit workflow: Projekt-Scan -> Berechnungslogik-Audit -> Plausibilitaetschecks -> Audit-Report
4. Critical rules:
   - Tilgung ist KEIN steuerlicher Abzug, nur Zinsen
   - AfA ist steuerlich abzugsfaehig, aber kein Cashflow-Abfluss
   - Grenzsteuersatz verwenden, nicht Durchschnittssteuersatz
   - Cent-Genauigkeit, keine Floating-Point-Fehler
