# MLP Design Guide V3

Use this skill whenever an output must follow MLP corporate design.

## Trigger

Activate this skill when the user mentions terms like:
- MLP Design
- Design Guide
- MLP Farben
- MLP CI
- MLP Corporate Design
- MLP CI/CD
- MLPDesignGuideV3
- MLP-konform

## Primary Reference

Read this file first:
- `references/MLPDesignGuideV3.md`

## Workflow

1. Identify output type (dashboard, website, presentation, slide, document, training material).
2. Apply mandatory MLP basics from the reference:
- Primary heading color `#033D5D`
- Divider/border structure color `#BEB6AA`
- Body text color `#2B2B2B` (never pure black)
- 8px spacing system
- Body text minimum 16px
3. Apply accessibility rules (WCAG contrast, no color-only status signaling).
4. Use subbrand colors only if audience requires it:
- MLP Med for medical audience
- SOFE for training/education
- Do not mix MLP Med and SOFE in one deliverable.
5. Before finishing, run a compact compliance check against the reference.

## Output Note Requirement

When delivering design work, include a short compliance note covering:
- palette used
- typography minimums
- spacing grid usage
- accessibility checks
- subbrand choice (or none)
